# 优化实现边界

本项目的性能提升必须来自静态程序分析和编译器代码变换，禁止在编译阶段解释执行或 JIT 执行整个 ToyC 程序，禁止预先计算 `main()` 的最终返回值后只生成返回常数的汇编。

当前采用的优化包括：

- 局部和稀疏条件常量传播、常量折叠；
- 复制传播、代数化简、公共子表达式消除；
- 死定义、死存储和不可达代码删除；
- 基本块与控制流图分析；
- 循环不变量外提、归纳变量强度削弱；
- 小函数内联和尾递归循环化；
- 活跃变量分析、图着色寄存器分配；
- RISC-V 指令选择和窥孔优化。

这些优化只根据 AST、IR、控制流图和数据流格值推导局部或过程内事实。常量求值仅适用于语言规定的 `const` 初始化、全局静态初始化和操作数均已证明为常量的表达式；函数调用、循环和具有副作用的操作不会通过执行程序来求值。

测试目录中的 `Riscv_sim` 只用于编译完成后的回归测试，不会被生产编译器链接或调用。生产入口固定为：

```text
源码 -> 语义检查 -> IR -> 静态优化 -> RISC-V 汇编
```

回归测试必须覆盖带函数调用和全局副作用的程序，确认 `-opt` 输出仍保留必要的运行时存储与读取，不能退化成 `main` 直接返回预计算常数。
